# V3.4 Planner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a V3.4 Planner-only learning version that turns a user question into structured plan JSON through FastAPI Swagger and CLI.

**Architecture:** V3.4 is a separate package under `obsidian_rag/v3_4/`. It uses LangGraph nodes for `build_prompt`, `call_planner`, and `parse_plan`; it reuses the existing OpenAI-compatible chat client, but does not execute retrieval tools yet.

**Tech Stack:** Python, Pydantic, FastAPI, OpenAI-compatible chat completions, pytest, SVG docs.

## Global Constraints

- Default answer language: Chinese.
- Commands, file names, config keys, and API names stay in English.
- V3.4 must have its own package, Swagger app, CLI command, tests, docs, and SVG diagrams.
- V3.4 Planner only plans; it must not execute RAG retrieval.
